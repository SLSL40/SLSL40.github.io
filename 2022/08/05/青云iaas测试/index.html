在测试环境上部署青云iaas平台

  view

—1.准备工作，在两台DELL R730 两台DELL R710上面进行部署 其中两台30是部署计算节点，
两台10是管理节点。准备U盘做系统引导盘，最好把提前准备好的网卡模板，还有settings和variables模板 还有青云的部署包
还有青云的包一起拷进来

—2.给服务器做RAID 不通厂家的服务器进入boot都不一样，按F11 ctrl+R进入RAID配置界面根据提前规划
好的方案进行配置，保存后ctrl+alt+delete重启服务器，然后进入BIOS界面选择启动盘安装系统（建议多带
几个引导盘，一次性多装几台加快进度）
—3.系统装完后，按照之前规划的ip和主机名来修改主机名和配置IP网卡

bash刷新主机名，拔U盘的时候记得umount卸载U盘

—4.做密钥，防止被T掉，把密码登录打开

vi /etc/ssh/sshd_config no改成yes

剩下的可以远程配置

lsblk检查一下分区，把多余的不能用的删掉

将部署包放置在root目录并解压: tar -zxf /root/qingcloud-installer_4.X.X-XXXX_amd64.tar.gz -C /root/

根据内置文档docs/qingcloud/how_to_deploy_qingcloud_4.X.md 来进行部署

—5.部署firstbox节点 根据之前规划好的方案

/root/qingcloud-installer/bootstrap/bootstrap_allinone.sh -m vlan210 -a 192.168.210.30

确保以上都没问题后

通过ip地址进入fristbox

建立ip_list文件，放入带部署节点的IP，每一行一个IP地址: vim /root/ip_list。

添加新节点: /pitrix/cli/add-nodes.py -I /root/ip_list

查看新添加的节点: /pitrix/cli/describe-nodes.py -s new

开始收集硬件信息: /pitrix/cli/collect-nodes.py -I /root/ip_list。

查看收集节点的当前状态, 使用: /pitrix/cli/get-collect-nodes-status.py。

查看收集节点的简略日志, 使用: /pitrix/cli/get-collect-nodes-log.py

详细日志文件为: /pitrix/log/node/collect.log

确保信息与服务器信息一致

在正式部署之前，该收集程序可以重复执行以修正硬件信息的错误, 正式部署之后，建议不要再次执行，如有硬件信息错误需要手动去数据库修改。

校验硬件信息:

获取所有节点的详细信息: /pitrix/cli/describe-nodes.py，查看各个节点的硬件信息

—6.配置青云平台

拷贝实现准备好的 variables.ZONE_ID.yaml 和settings.ZONE_ID.yaml

一定要确保这两个配置文件没有问题

生成variables文件及settings文件:

/pitrix/cli/config-qingcloud.py -v ‘/pitrix/config/variables.ZONE_ID.yaml’ -s ‘/pitrix/config/settings.ZONE_ID.yaml’`

查看配置QingCloud任务的当前状态，使用: /pitrix/cli/get-config-qingcloud-status.py。

查看配置QingCloud任务的简略日志，使用: /pitrix/cli/get-config-qingcloud-log.py。

校验variables和settings文件:

/pitrix/conf/variables/variables.yaml中的配置是否合理合法，并且必要的变量不能为空。

/pitrix/conf/settings里的文件要符合配置预期，可以按需修改部分字段，使之更加匹配安装要求

—7.部署青云平台

部署青云平台: /pitrix/cli/deploy-qingcloud.py -I /root/ip_list

查看部署青云平台的当前状态：/pitrix/cli/get-deploy-qingcloud-status.py。

查看部署青云平台的简略日志：/pitrix/cli/get-deploy-qingcloud-log.py。

step0 - step5一共五部全完成才算成功

通过如下命令检查环境(简略):

/pitrix/check/check.py -n all

cat /pitrix/check/formatted_result | egrep ‘Abnormal|Unknown|error’

将青云images拷贝到环境seed服务所在的节点，目录为/pitrix/images-repo。只需同步到任意一台seed节点即可，平台会自动同步镜像到所有的seed节点
